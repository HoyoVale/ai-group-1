# ai-group-1

这是 `hoyo ho` 的 AI 团队本地开发仓库。

仓库目标：

- 让 Butler 团队在本地协作开发多个项目
- 以一个仓库统一管理项目目录、共享资源和文档
- 由本地 git 工作流负责版本控制，再按需要推送到 GitHub

## 建议结构

```text
projects/   各个独立项目
shared/     可复用脚本、模板、公共资源
docs/       仓库级文档、规划、决策记录
```

## AI 团队分工

- Butler：拆解任务、分派员工、验收结果、汇总交付
- Atlas：研究、资料比较、方案摸底
- Dex：代码实现、脚本开发、排障修复
- Rowan：文档、说明、对外文稿
- Quinn：状态整理、任务跟进、下一步清单

## 建议工作方式

1. 每个新项目放在 `projects/<project-slug>/`
2. 共享逻辑优先放到 `shared/`
3. 文档和决策记录放到 `docs/`
4. 并行开发时优先用分支或 `git worktree`
5. 不要把无关项目混进同一个目录
